# OmenCore v3.7.0 RC Validation

Use this checklist for final release sign-off. Attach the generated logs, screenshots, and Model Identity text to the release issue or Discord test thread.

## Windows OMEN 16-xd0xxx / ProductId 8BCD

1. Open OmenCore as Administrator.
2. Copy **Settings > Model Identity Summary**.
3. In **General**, cycle `Balanced -> Performance -> Quiet -> Balanced` and confirm the active card, sidebar labels, tray label, and OSD agree.
4. Hold load for at least 5 minutes in Performance and confirm the fan curve reaches the restored high-temperature path near `75C`.
5. Press `Ctrl+Shift+E` and confirm profile cycling works without announcing Custom unless a custom curve exists.
6. Export diagnostics and include `runtime-performance.txt`, fan telemetry, and the app log.

## Windows OMEN 16-am0xxx / ProductId 8D2F

1. Copy **Model Identity Summary** and confirm resolved model `OMEN 16-am0xxx (8D2F)` with high confidence.
2. Apply General `Performance`, then `Balanced`, then `Performance` again.
3. Confirm logs show WMI thermal/performance-policy fallback when direct CPU PL1/PL2 values are unavailable or `0W/0W`.
4. Export diagnostics after the second Performance apply.

## Windows OMEN MAX 16-ah0xxx / ProductId 8D41

1. Copy **Model Identity Summary** and confirm ProductId `8D41` no longer reports keyboard model unknown.
2. Set GPU Power Boost to `Maximum`, start a repeatable GPU-heavy workload, and record the baseline GPU wattage.
3. In **General**, apply `Quiet`, `Balanced`, then `Performance`.
4. Without manually toggling Max fan, confirm the RTX 5080/5090 Laptop GPU can return above the previous roughly `100W` stuck state.
5. Open RGB/Lighting and confirm the page reports a per-key-capable MAX keyboard path or a backend-specific unavailable state, not generic laptop unsupported.
6. Export diagnostics and include the app log around all profile and lighting attempts.

## Windows Victus 16-s0xxx / RTX 3050

1. Confirm GPU Power Boost remains unavailable when firmware does not expose HP WMI TGP/PPAB control.
2. Open NVIDIA Tuning and confirm Power Limit is locked if NVAPI/firmware rejects writes.
3. Confirm core and memory clock offsets can still apply when supported by the driver.
4. Export diagnostics after one rejected power-limit attempt and one clock-only apply.

## Linux OMEN 15-en0xxx / Board 8787

Run:

```bash
sudo ./omencore-cli diagnose --report > omencore-8787-report.txt
sudo dmidecode -t system -t baseboard -t bios > dmidecode-8787.txt
find /sys/devices/platform -maxdepth 4 -iname '*profile*' -o -iname '*fan*' > hp-wmi-8787-paths.txt
dmesg | grep -Ei 'hp[-_]wmi|omen|platform_profile|WQ00' > kernel-8787-hp-wmi.txt
```

Confirm whether `ec_sys`, hp-wmi, or platform-profile paths physically change fan behavior. Do not treat missing RPM readback as failure by itself.

## Linux Victus 15-fb1xxx / Board 8C30

Run on Linux Mint 22.3 / Ubuntu 24.04 kernel 6.14 or newer:

```bash
sudo ./omencore-cli diagnose --report > omencore-8c30-report.txt
sudo dmidecode -t system -t baseboard -t bios > dmidecode-8c30.txt
find /sys/devices/platform -maxdepth 4 -iname '*profile*' -o -iname '*fan*' -o -iname '*pwm*' > hp-wmi-8c30-paths.txt
find /sys/class/hwmon -maxdepth 3 -type f > hwmon-8c30-files.txt
dmesg | grep -Ei 'hp[-_]wmi|victus|platform_profile|thermal_profile|WQ00' > kernel-8c30-hp-wmi.txt
```

The release claim remains diagnose-only unless the report proves a writable performance/profile path exists.
